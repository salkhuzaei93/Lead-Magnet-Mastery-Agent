# Quick Start Guide

## Your First Conversation

After setting up the project (see PROJECT_SETUP.md), open a new conversation and use one of these openers:

---

### Opener A — Starting from Scratch

```
I want to build a lead magnet for [your market].

Here's what I know:
- Market: [e.g., "freelance copywriters, 1–3 years experience, struggling to find premium clients"]
- Problem the lead magnet addresses: [e.g., "they don't know how to position as a specialist"]
- Paid offer it bridges to: [e.g., "6-week positioning coaching program at $1,500"]
- Existing research: [either "none" or paste what you have]

Begin with Skill 01.01.
```

---

### Opener B — You Have Existing Research

```
I want to build a lead magnet and I have some research already.

Market: [market]
Existing data I have:
[Paste Reddit threads, Amazon reviews, competitor pages, customer interviews, etc.]

Process this through Phase 01 skills. Tell me which skills you can complete from this data and which gaps I need to fill.
```

---

### Opener C — Resuming a Session

```
I'm continuing a lead magnet project. Here are the outputs from our last session:

[Paste the output artifacts from the previous conversation]

Pick up from Skill [XX.XX].
```

## Shortcut Commands

These work inside a running project conversation:

| Command | What It Does |
|---------|-------------|
| `"Run skill 01.01"` | Executes that specific skill |
| `"What outputs do we have so far?"` | Lists all completed skill outputs |
| `"What's missing before we can advance?"` | Shows which skills are incomplete |
| `"Summarize the Prospect Portrait"` | Recalls the 01.06 output |
| `"What's our Promise Statement?"` | Recalls the 02.01 output |
| `"Check the Vagueness Watchlist"` | Activates 02.02 audit on any new copy |
| `"Run the Value-Friction Calculator"` | Executes the 02.03 scoring tool |
| `"What format is recommended?"` | Recalls the 02.04 output |
| `"Show the Bridge Blueprint"` | Recalls the 02.05 output |
| `"What's our quick win?"` | Recalls the 02.07 specification |

## The Fastest Useful Output

If you're short on time and want to understand what the agent can do before committing to a full project, run just these three skills in one session:

1. **01.01** — Awareness Level Diagnosis (15 minutes, requires answering 7 questions)
2. **01.06** — Psychographic Profiling (20 minutes, produces the Copy Anchor Statement)
3. **02.01** — One Big Promise (15 minutes, produces a scorecarded Promise Statement)

These three outputs alone give you: the right awareness entry angle, a fully imagined prospect portrait to write from, and a tested promise statement. That's enough to write significantly better opt-in page copy than most lead magnets in your market.

The remaining 11 skills make the whole system airtight — but these three are the highest-leverage starting point.

## Common Questions Before Starting

**"Do I need to complete every skill before building my lead magnet?"**
For Phase 01: yes, all 6 are required. Skipping market intelligence is how you end up with a well-written lead magnet for the wrong promise.
For Phase 02: skills 02.01–02.05 are required. Skills 02.06–02.08 are strongly recommended.

**"How much research do I need to bring in?"**
For Skill 01.03 (Conversation Archaeology), the more raw market language you paste in — Reddit threads, reviews, comments — the better the VOM Report. For the other skills, the agent can work from your answers to its questions. Bringing research in speeds up Phase 01 significantly.

**"Can I run the agent on a lead magnet I've already built?"**
Yes. Paste the existing lead magnet into the conversation and tell the agent: "Audit this lead magnet against the Phase 01 and Phase 02 frameworks. Identify which outputs are missing and what needs to be fixed." It will run the skills as an audit rather than as a build-from-scratch process.

**"What if I disagree with the agent's recommendation?"**
Push back with specifics. The agent will explain the framework basis for its recommendation. If you have market-specific information that overrides the framework's default recommendation, tell the agent — it will incorporate the override and note it in the output. The frameworks are defaults built on pattern recognition across thousands of cases, not absolute rules.
